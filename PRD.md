## 1. 项目概述

### 1.1 项目背景

随着企业 IT 基础设施的规模化扩张，运维人员需要频繁在多台服务器上安装、配置和管理中间件应用（如 MySQL、Redis、Elasticsearch 等）。传统的手动操作方式存在以下痛点：

- **效率低下**：每台服务器需手动登录、下载、安装、配置
- **容易出错**：人工操作易遗漏配置项或输入错误参数
- **难以追溯**：安装过程缺乏统一的日志记录和版本管理
- **学习成本高**：不同应用的安装步骤复杂，新人上手困难

### 1.2 项目目标

开发一款基于 **C# + WPF** 的跨平台服务器管理工具，实现以下目标：

- ✅ **一键安装**：通过图形化界面完成远程应用的自动化安装
- ✅ **跨平台支持**：支持 Windows Server、CentOS、Ubuntu 三种操作系统
- ✅ **实时监控**：实时查看安装进度和日志输出
- ✅ **统一管理**：集中管理多台服务器的连接信息和应用状态

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| UI 框架 | WPF (.NET 8) |
| MVVM 框架 | CommunityToolkit.MVVM |
| UI 组件库 | MaterialDesignInXaml |
| 远程通信 | Renci.SshNet (SSH/SFTP) |
| 本地存储 | SQLite |
| 加密 | AES-256 |

### 1.4 支持的应用列表

| 应用 | 版本 | Linux | Windows | 说明 |
|------|------|-------|---------|------|
| MySQL | 8.x | ✅ | ✅ | 关系型数据库 |
| Elasticsearch | 8.x | ✅ | ✅ | 搜索引擎，需 JDK 17+ |
| Redis | 7.x | ✅ | ✅ | 内存数据库 |
| RabbitMQ | 3.12.x | ✅ | ✅ | 消息队列，需 Erlang 25+ |
| Nacos | 2.3.x | ✅ | ✅ | 服务注册中心，需 JDK 8+ |
| Nginx | 1.24.x | ✅ | ✅ | Web 服务器/反向代理 |

### 3.2 功能详情

#### F-001 ~ F-004: 主机管理

**功能描述**: 管理远程服务器的连接信息

**输入**:
- 主机名称（文本，必填）
- IP 地址（IP 格式校验，必填）
- 端口（数字，默认 22，范围 1-65535）
- 用户名（文本，必填）
- 认证方式：
  - 密码（密码框，必填）
  - 私钥路径（文件选择，可选）
- 操作系统类型（下拉选择：Windows/CentOS/Ubuntu，必填）
- 分组名称（文本，可选）

**处理逻辑**:
1. 表单校验通过后保存
2. 密码使用 AES-256 加密后存储到 SQLite
3. 测试连接时建立 SSH 会话并验证
4. 定期检测在线状态（心跳检测）

**输出**:
- 主机列表（名称、IP、状态图标、OS 图标）
- 连接测试结果（成功/失败及原因）

#### F-005 ~ F-007: 应用市场

**功能描述**: 展示可安装的应用及其状态

**输入**:

**处理逻辑**:

2. 对每个应用检测在目标主机的安装状态
3. 根据 OS 类型过滤不支持的应用

**输出**:
- 应用卡片网格布局
- 每个卡片包含：
  - 应用图标
  - 应用名称
  - 版本号
  - 简短描述
  - 支持平台图标

#### F-010 ~ F-013: 安装执行

**功能描述**: 执行完整的远程安装流程

**流程**:
```
1. 检测连接状态 → 2. 获取安装包 → 3. 上传到远程
       ↓
5. 验证安装结果 ← 4. 执行安装脚本
```

**输入**:
- 目标主机
- 应用信息
- 配置参数
- 安装包来源（本地路径/仓库）

**处理逻辑**:
1. **连接检测**: 验证 SSH 连接是否可用
2. **获取安装包**: 
   - 本地：读取本地文件路径
   - 仓库：从配置的安装包仓库下载
3. **上传文件**: 使用 SFTP/SCP 上传到远程临时目录
4. **执行脚本**: 
   - 根据 OS 类型选择对应脚本
   - 配置参数作为环境变量传递
   - 实时捕获脚本输出
5. **验证结果**: 执行检测脚本确认安装成功

**输出**:
- 任务 ID
- 进度事件流
- 日志事件流
- 最终结果（成功/失败）

---

#### F-014 ~ F-016: 进度监控

**功能描述**: 实时显示安装进度和日志

**进度上报格式**（脚本输出）:
```
PROGRESS:阶段名称:百分比
```

**日志级别**:
- `INFO` - 一般信息（蓝色）
- `SUCCESS` - 成功信息（绿色）
- `WARNING` - 警告信息（黄色）
- `ERROR` - 错误信息（红色）

**UI 组件**:
- 进度条（百分比 + 阶段名称）
- 日志查看器（可滚动、自动滚动到底部）

---

#### F-017 ~ F-019: 应用检测

**功能描述**: 检测应用在远程主机的状态

**检测项**:
1. **是否安装**: 检查安装目录或进程
2. **版本号**: 执行版本命令或读取版本文件
3. **运行状态**: 检查服务状态
4. **端口占用**: 使用 netstat/ss 命令检查

**Linux 检测脚本示例**:
```bash
#!/bin/bash
# check-installed.sh

# 检测 MySQL
if command -v mysql &> /dev/null; then
    echo "INSTALLED:true"
    echo "VERSION:$(mysql --version | grep -oP '\d+\.\d+\.\d+')"
    echo "RUNNING:$(systemctl is-active mysqld)"
    echo "PORT:$(netstat -tlnp | grep mysqld | grep -oP '\d+$')"
else
    echo "INSTALLED:false"
fi
```

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 启动时间 | ≤ 3 秒 | 从点击图标到主界面可用 |
| 连接建立 | ≤ 5 秒 | SSH 连接建立时间 |
| 文件上传 | ≥ 1MB/s | 100M 网络环境下 |
| 并发任务 | 可配置（默认 3） | 同时执行的最大任务数 |
| 内存占用 | ≤ 500MB | 空闲状态下 |
| 日志轮转 | 单文件 ≤ 10MB | 超过后自动分割 |

### 4.2 安全需求

| 要求 | 实现方式 |
|------|----------|
| 密码加密存储 | AES-256 加密，密钥从安全位置获取 |
| 传输加密 | SSH 协议自带加密（Linux）、WinRM over SSL（Windows） |
| 操作审计 | 记录所有安装/卸载操作到日志 |
| 权限控制 | 仅允许授权用户操作（未来扩展） |
| 私钥保护 | 支持带密码保护的私钥文件 |

### 4.3 可靠性需求

| 要求 | 实现方式 |
|------|----------|
