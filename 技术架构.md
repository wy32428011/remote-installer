# 远程安装应用 - 技术架构说明

## 1. 文档定位

本文档描述的是当前仓库已经落地的技术架构，而不是早期设计稿或未来规划。

适用范围：
- 当前 `RemoteInstaller` WPF 客户端的真实模块划分
- 当前主窗口、主编排层、服务层、SQLite、本地日志和远程连接能力
- 当前配置来源、安装执行链路和安全机制

不包含：
- 尚未落地的抽象接口体系
- 未在代码中实现的性能承诺
- 立项阶段的未来规划与初始化步骤

---

## 2. 总体架构概览

当前程序是一个运行在 Windows 上的 `net10.0-windows` WPF 桌面应用，采用 MVVM 组织界面与状态，但业务编排并不是分散在多个独立领域 ViewModel 中，而是以 `MainViewModel` 为核心入口，由多个具体 Service 协作完成远程连接、安装执行、配置管理、任务记录和日志输出。

```text
┌─────────────────────────────────────────────────────────────┐
│ WPF 桌面客户端（MainWindow）                                │
├─────────────────────────────────────────────────────────────┤
│ View 层                                                     │
│ ├─ MainWindow                                               │
│ ├─ Dialogs（添加主机、安装配置、任务进度、日志、终端等）      │
│ └─ Controls（脚本管理、公共控件等）                         │
├─────────────────────────────────────────────────────────────┤
│ 主编排层                                                    │
│ └─ MainViewModel                                            │
│    ├─ 主机列表与选择状态                                    │
│    ├─ 应用市场与筛选                                        │
│    ├─ 批量检测 / 批量安装 / 批量卸载                         │
│    ├─ 任务列表与底部日志                                    │
│    ├─ 自定义应用与脚本管理入口                              │
│    └─ 主题、设置、历史记录等界面状态                        │
├─────────────────────────────────────────────────────────────┤
│ 服务层                                                      │
│ ├─ SshService                                               │
│ ├─ InstallerService                                         │
│ ├─ ConfigurationService                                     │
│ ├─ AppConfigurationService                                  │
│ ├─ CustomApplicationService                                 │
│ ├─ DatabaseService                                          │
│ ├─ LoggerService / FileLoggerService                        │
│ ├─ EncryptionService                                        │
│ └─ HostStatusRefreshCoordinator                             │
├─────────────────────────────────────────────────────────────┤
│ 本地数据与外部依赖                                          │
│ ├─ SQLite（data.db）                                        │
│ ├─ Scripts/app-configuration.json                           │
│ ├─ 本地日志目录 %LocalAppData%\RemoteInstaller\Logs        │
│ └─ SSH.NET（SSH / SFTP / Shell）                            │
└─────────────────────────────────────────────────────────────┘
```

## 3. 技术栈与运行基础

### 3.1 客户端与 UI

| 类别 | 当前实现 | 说明 |
|------|----------|------|
| 目标框架 | `net10.0-windows` | 来自 `RemoteInstaller.csproj` |
| UI 框架 | WPF | Windows 桌面客户端 |
| MVVM | CommunityToolkit.MVVM | `ObservableObject`、`RelayCommand` 等 |
| UI 风格 | MaterialDesignThemes | 主题与控件样式 |

### 3.2 远程连接与文件传输

| 能力 | 当前实现 | 说明 |
|------|----------|------|
| 远程连接 | SSH | `SshService` 统一处理 |
| 文件传输 | SFTP | 用于安装包、脚本和远程文件面板等需要文件能力的场景 |
| 终端交互 | ShellStream | 用于远程终端实时交互 |
| Windows 目标机 | SSH | 当前不是以 WinRM 为主路径 |

### 3.3 本地存储与配置

| 类型 | 当前实现 | 说明 |
|------|----------|------|
| 本地数据 | SQLite | 主机、任务、日志、历史、自定义应用、设置 |
| 应用市场配置 | JSON | `Scripts/app-configuration.json` |
| 系统设置 | SQLite `settings` 表 | 仓库、代理、超时、并发、主题等 |
| 本地缓存 | 文件系统目录 | 默认 `%LocalAppData%\RemoteInstaller\Cache` |

### 3.4 安全机制

| 类型 | 当前实现 | 说明 |
|------|----------|------|
| 凭据保护 | Windows DPAPI | `EncryptionService` 使用 `ProtectedData` |
| SSH 认证 | 密码 / 私钥 | 由 `SshService` 构造连接信息 |
| 日志 | 文件滚动写入 | `FileLoggerService` 控制输出与滚动 |

## 4. UI 层与对象装配

### 4.1 主窗口结构

`MainWindow.xaml` 是当前程序的统一交互入口，主界面集中承载以下区域：
- 顶部操作按钮
- 左侧服务器列表
- 中间标签页（应用市场 / 自定义应用 / 脚本管理）
- 右侧任务面板
- 底部日志区
- 状态栏与主题切换相关状态

`MainWindow.xaml.cs` 负责少量窗口级交互逻辑，例如：
- 首屏加载后调用 `MainViewModel.InitializeAfterWindowLoadedAsync()`
- 服务器多选同步到 ViewModel
- 双击任务打开任务进度窗口
- 根据窗口宽度在左右分栏与上下堆叠布局之间切换

### 4.2 当前对象装配方式

当前项目虽然引用了 `Microsoft.Extensions.DependencyInjection`，但主窗口的数据上下文装配主要通过 `Locator.Instance` 完成。`Locator.cs` 会按单例方式创建：
- `LoggerService`
- `DatabaseService`
- `SshService`
- `ConfigurationService`
- `HostStatusRefreshCoordinator`
- `AppConfigurationService`
- `MainViewModel`

这意味着当前架构更接近“主窗口 + 单核心 ViewModel + 多具体服务协作”的实用型结构，而不是多层接口和多领域 ViewModel 并行装配的设计稿结构。

### 4.3 应用启动与主题

`App.xaml.cs` 负责应用启动级初始化，包括：
- WPF 渲染/DPI 相关初始化
- UTF-8 编码注册，减少日志乱码
- 通过 `App.SwitchTheme` 切换 Material Design 主题与自定义资源字典

---

## 5. 主编排层：MainViewModel

`RemoteInstaller/ViewModels/MainViewModel.cs` 是当前最重要的业务编排入口，负责串联界面状态、后台服务和用户操作。

其核心职责包括：
- 维护主机列表、选中主机和服务器过滤状态
- 维护应用市场、自定义应用、任务列表和底部日志集合
- 驱动批量检测、批量安装、批量卸载等操作
- 协调主题、设置、历史记录、终端、日志查看器等入口
- 在首屏加载完成后补充加载自定义应用等后台初始化内容

当前几个关键事实：
- 批量安装/卸载要求“已选服务器 + 已选应用 + 当前无活动批量任务”
- 批量检测只要求选择服务器，不要求先勾选应用
- 任务、过滤和日志都由 `MainViewModel` 统一维护，而不是拆成多个独立任务中心

因此，后续阅读代码时可以将 `MainViewModel` 视为当前桌面客户端的“主流程编排器”。

## 6. 关键服务模块说明

### 6.1 SshService

`SshService` 是远程基础能力中心，负责：
- 连接远程主机
- 测试 SSH 连通性
- 检测远程操作系统类型与版本
- 支持密码认证和私钥认证
- 执行远程命令
- 上传/下载文件与目录
- 提供终端 ShellStream 能力

当前无论目标主机是 Linux 还是 Windows，主路径都是 SSH，不应再把 WinRM 作为当前架构的中心能力来描述。

### 6.2 InstallerService

`InstallerService` 负责安装、卸载、检测等流程编排，典型职责包括：
- 创建安装任务对象并落库
- 连接目标主机
- 创建远程工作目录
- 上传安装脚本
- 上传本地文件或目录型离线资源
- 结合 `LogCollector` 解析脚本输出与进度
- 将任务进度和日志同步回界面与数据库

它是“应用执行链路”的核心服务，而不是一个简单的脚本调用包装器。

### 6.3 ConfigurationService

`ConfigurationService` 提供远程配置文件管理能力，负责：
- 根据应用名称和操作系统查找配置文件路径
- 读取远程配置
- 备份远程配置
- 保存更新后的配置内容

它支撑的是当前程序中的远程配置运维场景，不应再写成抽象的“配置中心”。

### 6.4 AppConfigurationService

`AppConfigurationService` 负责从 `Scripts/app-configuration.json` 读取应用市场定义，并提供：
- 应用列表读取
- 版本列表读取
- 参数读取
- 安装/卸载/检测脚本路径解析
- 分类读取
- 配置文件热刷新

该服务通过 `FileSystemWatcher` 监听配置文件变化，应用市场是“配置驱动”的，而不是在代码中写死固定应用清单。

### 6.5 CustomApplicationService

`CustomApplicationService` 负责自定义应用相关能力，包括：
- 上传本地文件或目录到远程目录
- 启动自定义应用
- 读取远程配置内容
- 保存远程配置内容

这说明“自定义应用”不是外围补充功能，而是已经进入当前正式架构边界的能力。

### 6.6 DatabaseService

`DatabaseService` 负责 SQLite 初始化与持久化操作，是本地数据中心。它会在 `%LocalAppData%\RemoteInstaller` 创建和维护 `data.db`，并初始化主表结构；如果检测到安装目录中的旧版数据库，会在首次启动时自动迁移。

### 6.7 LoggerService / FileLoggerService

- `LoggerService` 负责应用运行过程中的日志汇总
- `FileLoggerService` 负责日志写入文件、定时刷新和滚动归档

当前文件日志目录为：
- `%LocalAppData%\RemoteInstaller\Logs`

### 6.8 EncryptionService

`EncryptionService` 使用 Windows DPAPI：
- `ProtectedData.Protect`
- `ProtectedData.Unprotect`
- `DataProtectionScope.CurrentUser`

当前凭据保护方案是“基于当前用户上下文的 DPAPI”，不是固定密钥 AES 示例。

### 6.9 HostStatusRefreshCoordinator

`HostStatusRefreshCoordinator` 负责主机状态刷新过程中的：
- 状态快照缓存
- 防抖
- 并发中的刷新去重
- 失效控制

因此，当前主机状态刷新并不是简单的同步轮询，而是带有前端交互优化的协调机制。

## 7. 数据存储与配置来源

### 7.1 SQLite 数据库

`DatabaseService` 当前初始化的主要表包括：
- `hosts`
- `tasks`
- `settings`
- `logs`
- `install_history`
- `custom_apps`

几个重要事实：
- 主机主键是字符串 ID，不是旧文档中的自增整数
- 任务、日志和安装历史都已经有独立存储
- 自定义应用定义也会进入数据库持久化

数据库文件位置：
- `%LocalAppData%\RemoteInstaller\data.db`

### 7.2 系统设置

`SystemSettings` 定义了当前系统级配置字段，主要包括：
- 仓库地址与 Token
- 代理设置
- SSH 超时、重试次数、重试间隔
- 缓存目录与缓存大小
- 最大并发任务数
- 当前主题

默认值示例：
- `ConnectionTimeout = 60`
- `RetryCount = 3`
- `RetryInterval = 5`
- `MaxConcurrentTasks = 3`
- `CacheDirectory = %LocalAppData%\RemoteInstaller\Cache`

### 7.3 应用市场与脚本来源

应用市场并非由代码中的固定枚举维护，而是来自：
- `Scripts/app-configuration.json`
- 配套安装/卸载/检测脚本资源目录

`AppConfigurationService` 会解析当前应用、版本、参数和脚本路径，因此应用支持能力应始终以配置文件与当前程序界面显示为准。

### 7.4 日志存储

当前存在两类日志：
- 界面内存日志：用于主窗口底部日志区、任务日志与实时展示
- 文件日志：用于落盘和问题排查

`FileLoggerService` 使用队列 + 定时刷新方式写文件，并在超过大小阈值后做滚动归档。

## 8. 典型运行链路

### 8.1 启动链路

1. 应用启动时由 `App.xaml.cs` 完成基础初始化和编码设置。
2. `MainWindow` 创建后，将 `DataContext` 绑定到 `Locator.Instance`。
3. `Locator` 初始化核心服务并创建 `MainViewModel`。
4. `MainViewModel` 加载设置、主机、筛选项和基础界面状态。
5. 主窗口加载完成后，调用 `InitializeAfterWindowLoadedAsync()` 执行首屏后的补充初始化，如加载自定义应用。

### 8.2 主机状态刷新链路

1. 用户选择主机或触发批量检测。
2. `MainViewModel` 发起状态刷新请求。
3. `HostStatusRefreshCoordinator` 做缓存命中判断、防抖和并发去重。
4. `SshService` 执行远程检测。
5. 结果回写到主机状态、界面显示和相关日志。

### 8.3 安装执行链路

1. 用户在应用市场选择应用和目标主机，填写参数。
2. `MainViewModel` 触发安装命令。
3. `InstallerService` 创建任务并保存到数据库。
4. `SshService` 连接目标主机并创建远程工作目录。
5. 上传安装脚本和本地离线资源。
6. 远程脚本执行过程中，`LogCollector` 解析 `PROGRESS:阶段:百分比` 输出。
7. 任务进度、日志和状态同步更新到界面与数据库。
8. 安装完成后保留任务和历史记录，供后续查看。

### 8.4 远程配置链路

1. 用户打开配置入口。
2. `ConfigurationService` 根据应用和操作系统推断配置文件候选路径。
3. 通过 `SshService` 读取远程配置内容。
4. 用户修改后再通过 `ConfigurationService` 写回目标主机。

## 9. 安全与认证机制

### 9.1 本地敏感信息保护

主机密码、密钥口令等敏感信息不会明文直接持久化。当前实现使用 Windows DPAPI，并绑定到当前用户上下文。

这意味着：
- 本地保存的敏感字段经过加密保护
- 解密依赖当前用户环境
- 文档中不应再描述为固定密钥的 AES 对称加密方案

### 9.2 SSH 认证能力

`SshService` 当前支持：
- 用户名 + 密码认证
- 用户名 + 私钥认证
- 私钥口令解密后加载

因此，当前远程认证边界是标准 SSH 认证模型。

## 10. 当前能力边界与非目标

为了避免文档再次漂移，下面这些内容不应被描述为“当前已经实现”：
- 以抽象接口和多层拆分为中心的理想化服务架构
- 当前代码中并不存在的历史设计类或占位模块
- 单一静态配置文件作为全部主配置来源
- 固定密钥的对称加密方案
- 非 SSH 的远程能力作为当前主路径
- 分片上传、断点续传等未被当前代码明确证实的性能承诺
- 项目初始化路线图、后续研发计划或“确认后开始实现”的阶段性说明

当前文档应聚焦“现在代码如何工作”，而不是“未来也许怎样演进”。
