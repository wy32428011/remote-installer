# RemoteInstaller 用户使用手册

## 1. 软件简介

RemoteInstaller 是一款运行在 Windows 上的桌面工具，用于通过 SSH 方式远程连接服务器，并完成应用安装、卸载、状态检测、终端运维、脚本管理和自定义应用部署等工作。

本文档面向最终使用人员，重点说明当前程序界面中可以直接完成的操作流程。

> 说明
>
> - 本手册以当前程序界面和当前代码实现为准。
> - 若本手册与旧版 README、历史截图或旧培训材料不一致，请以当前程序界面为准。
> - 应用市场中的应用、版本和安装参数由程序当前配置决定，实际显示内容以界面为准。

### 1.1 当前内置应用示例

当前应用市场中可见的内置应用示例包括：

- JDK 8
- JDK 11
- JDK 17
- MySQL
- MariaDB
- Redis
- Elasticsearch
- RabbitMQ
- Consul
- Traefik
- Nginx

> 说明：
> 以上为当前程序内置清单示例。具体可安装应用、版本和参数以“应用市场”页面实际显示为准。

## 2. 使用前准备

## 2.1 客户端要求

RemoteInstaller 是 Windows 桌面程序。

- 运行环境：Windows 10/11 或 Windows Server
- 若使用发布版，按发布包提供的方式直接启动即可
- 若从源码运行，当前项目目标框架为 `.NET 10 Windows`

## 2.2 目标主机要求

在使用本工具前，请确认目标主机满足以下条件：

- 与本机网络可达
- 已开启 SSH 服务
- 已知可登录的用户名和密码，或可用私钥
- 安装应用时具备足够权限
- Linux 主机建议使用具备 `sudo` 权限的账号

> 重要：
> 无论目标主机是 Linux 还是 Windows，当前程序都通过 SSH 进行远程连接和执行操作。对于 Windows 目标机，也需要提前启用并配置 OpenSSH 服务。

## 2.3 首次启动后的本地数据

程序首次启动后会自动创建本地数据文件和日志目录：

- 用户数据目录：`%LocalAppData%\RemoteInstaller`
- 数据库文件：`%LocalAppData%\RemoteInstaller\data.db`
- 日志目录：`%LocalAppData%\RemoteInstaller\Logs`
- 缓存默认目录：`%LocalAppData%\RemoteInstaller\Cache`

新版本首次启动时，如果安装目录中存在旧版 `data.db`，程序会自动迁移到用户数据目录；重新安装不会替换该目录下的现有数据库。

本地数据库主要用于保存：

- 已添加的服务器信息
- 任务记录
- 历史记录
- 系统设置
- 日志记录

## 2.4 敏感信息说明

- 服务器登录密码、私钥口令等敏感信息会加密后保存在本地
- 请妥善保管本机上的程序目录、日志目录和数据库文件
- 日志中可能包含服务器名称、路径、执行结果等运维信息

## 3. 快速开始

如果你是第一次使用，建议按以下顺序完成：

1. 启动程序
2. 添加一台服务器
3. 测试连接并保存
4. 在左侧选择该服务器
5. 在“应用市场”中选择一个应用执行安装
6. 在右侧任务列表中查看进度
7. 如需排查问题，打开任务详情或日志查看器

### 3.1 启动程序

启动后，主界面主要分为以下区域：

- 顶部操作按钮区
- 左侧服务器列表
- 中间功能标签页
- 右侧任务列表
- 底部操作日志
- 底部状态栏

### 3.2 添加第一台服务器

1. 点击顶部 **添加服务器**。
2. 在弹出的窗口中填写：
   - 主机名称
   - 主机地址
   - 端口
   - 用户名
   - 认证方式
3. 根据认证方式填写：
   - 密码认证：输入密码
   - 私钥认证：选择私钥文件
4. 点击 **测试连接**。
5. 测试成功后，点击 **保存**。

### 3.3 安装第一个应用

1. 在左侧服务器列表中选中目标服务器。
2. 在“应用市场”中找到目标应用。
3. 点击应用卡片上的 **安装**。
4. 在安装配置窗口中选择安装来源并填写参数。
5. 点击 **确认** 开始执行。
6. 在右侧任务列表中查看任务状态。
7. 如需查看详细过程，点击任务卡片中的 **详情**，或双击任务。

## 4. 主界面说明

## 4.1 顶部操作按钮区

主界面顶部提供以下主要入口：

| 按钮 | 作用 |
|------|------|
| 添加服务器 | 新增远程服务器 |
| 批量检测 | 对已选服务器批量刷新应用状态 |
| 批量安装 | 对已选服务器批量安装已选中的应用 |
| 批量卸载 | 对已选服务器批量卸载已选中的应用 |
| 终端 | 打开当前选中服务器的远程终端 |
| 自定义应用 | 进入自定义应用部署入口 |
| JDK 上传 | 打开 JDK 相关上传/安装入口 |
| 历史记录 | 查看安装/卸载历史 |
| 设置 | 打开系统设置窗口 |
| 日志 | 打开日志查看器 |
| 主题切换 | 切换浅色/深色主题 |

### 按钮启用规则

- **终端**、**自定义应用**、**JDK 上传**：需要先选中一台服务器
- **批量检测**：需要至少选中一台服务器
- **批量安装/批量卸载**：需要同时满足“已选服务器”和“已选应用”

## 4.2 左侧服务器列表

左侧用于管理服务器。

支持的功能包括：

- 搜索服务器
- 按分组筛选服务器
- 查看服务器名称、IP、操作系统、版本
- 多选服务器
- 编辑服务器
- 删除服务器

### 多选说明

服务器列表支持多选。批量检测、批量安装和批量卸载依赖多选结果。

## 4.3 当前服务器信息栏

当你选中一台服务器后，右侧上方会显示当前服务器信息，包括：

- 服务器名称
- IP 和端口
- 操作系统类型

你还可以点击 **刷新应用**，重新获取该服务器上的应用状态。

## 4.4 中间功能标签页

主界面中间区域包含三个主要标签页：

- 应用市场
- 自定义应用
- 脚本管理

## 4.5 右侧任务列表

右侧显示任务列表，用于查看：

- 当前任务状态
- 当前阶段
- 百分比进度
- 最近活动信息

支持按状态筛选任务，例如：

- 全部
- 进行中
- 失败
- 已完成
- 已取消

## 4.6 底部操作日志

底部日志区域用于显示程序操作过程中的实时日志，适合快速查看：

- 连接动作
- 安装过程
- 错误信息
- 执行结果

## 4.7 底部状态栏

状态栏显示：

- 已连接主机数
- 任务数
- 当前仓库地址

## 5. 主机管理

## 5.1 添加服务器

在“添加服务器”窗口中，需要填写以下信息：

- 主机名称
- 主机地址
- 端口
- 用户名
- 认证方式
- 密码或私钥路径
- 分组名称（可选）

### 认证方式

当前支持两种认证方式：

1. 密码认证
2. 私钥认证

### 操作系统识别

“操作系统类型”字段由程序在连接测试成功后自动识别，用户无需手动填写。

## 5.2 保存前必须先测试连接

添加或修改服务器时，必须先执行 **测试连接**。

只有满足以下条件时，**保存** 按钮才会可用：

- 基本信息填写完整
- 连接测试成功

如果连接测试失败，请先处理连接问题，再保存。

## 5.3 编辑服务器

在左侧服务器列表中，点击对应服务器卡片上的 **编辑** 按钮即可修改服务器信息。

## 5.4 删除服务器

在左侧服务器列表中，点击对应服务器卡片上的 **删除** 按钮即可移除服务器。

> 说明：
> 删除服务器只会移除本地保存的主机记录，不会对目标主机本身执行卸载或清理操作。

## 5.5 分组与搜索

你可以通过以下方式快速定位服务器：

- 在搜索框中输入关键字
- 通过分组下拉框筛选

分组名称由你在添加或编辑服务器时填写。

## 6. 应用市场

## 6.1 应用市场的作用

“应用市场”用于对目标服务器上的内置应用执行以下操作：

- 安装
- 状态刷新
- 配置
- 卸载
- 选中为批量操作目标

## 6.2 应用来源说明

应用市场中的应用、版本和安装参数由程序当前配置提供。

因此：

- 不同版本程序中应用数量可能不同
- 同一应用的版本列表可能变化
- 安装参数会随当前配置变化

使用时请始终以当前界面显示为准。

## 6.3 搜索与分类筛选

应用市场支持：

- 搜索应用名称
- 按分类筛选应用

如果需要对多种应用进行批量操作，可以先在应用卡片上逐个点击选中。

## 6.4 应用卡片信息说明

应用卡片通常显示：

- 应用图标
- 应用名称
- 当前版本
- 适用平台标记
- 安装状态
- 已安装版本（如已识别）

常见状态包括：

- 已安装
- 运行中

## 6.5 安装应用

1. 先在左侧选中服务器。
2. 在应用市场中找到目标应用。
3. 点击 **安装**。
4. 进入安装配置窗口。
5. 选择安装来源并填写参数。
6. 确认后开始执行。

## 6.6 刷新应用状态

点击应用卡片上的 **刷新**，可重新检测该应用在当前服务器上的状态。

适用场景：

- 安装完成后确认结果
- 手动修改服务后重新检测
- 卸载后确认是否已完全移除

## 6.7 配置应用

点击应用卡片上的 **配置**，可进入对应应用的配置查看或编辑流程。

适用场景：

- 查看远程配置文件
- 修改配置参数
- 保存配置，并在界面支持时执行后续生效操作

> 说明：
> 不同应用可编辑的配置内容、保存方式和后续生效操作可能不同，具体以弹出的配置界面为准。

## 6.8 卸载应用

点击应用卡片上的 **卸载**，程序会执行对应应用的卸载流程。

建议在卸载前确认：

- 当前服务器选择正确
- 当前应用选择正确
- 业务允许停止或移除该服务

## 6.9 安装来源说明

安装配置窗口中，应用通常支持以下一种或两种来源：

- 本地资源
- 在线安装

实际可选项以当前应用和当前界面显示为准。

### 常见说明

- 程序会优先支持从本地资源安装
- 某些应用支持在线安装
- 某些应用仅支持本地离线资源
- JDK 支持本地目录上传和离线资源使用

### MariaDB 特别说明

当前界面中已明确提示：

- MariaDB 当前仅支持 `Scripts/MariaDB` 下的离线资源安装

## 6.10 安装参数说明

不同应用的安装参数不同，常见参数包括：

- 端口
- 密码
- 路径
- 布尔开关
- 版本
- 安装目录
- 数据目录

参数窗口中带 `*` 的项目通常表示必填项。

## 7. 批量操作

## 7.1 批量检测

适用于：

- 同时刷新多台服务器的应用状态
- 批量重新获取各服务器上的应用安装与运行情况

使用方法：

1. 在左侧多选服务器。
2. 点击顶部 **批量检测**。
3. 程序会对所选服务器执行应用状态刷新。
4. 在右侧任务列表查看执行情况。

## 7.2 批量安装

使用方法：

1. 多选服务器。
2. 在应用市场中选中一个或多个应用。
3. 点击顶部 **批量安装**。
4. 程序会按队列逐个安装，同一时间只执行一个“服务器 + 应用”任务。
5. 在任务列表中查看各服务器上的安装进度。

## 7.3 批量卸载

使用方法：

1. 多选服务器。
2. 在应用市场中选中一个或多个应用。
3. 点击顶部 **批量卸载**。
4. 在任务列表中查看执行结果。

## 7.4 批量操作前提

批量安装和批量卸载必须同时满足：

- 已选择至少一台服务器
- 已在应用市场中选择至少一个应用
- 当前没有正在执行的批量任务

## 7.5 并发数量

批量检测和批量卸载的同时执行数量受“系统设置”中的最大并发任务数影响。

批量安装固定为全局串行执行，不受最大并发任务数影响，前一个安装任务结束后才会开始下一个安装任务。

## 8. 终端与远程文件管理

## 8.1 打开远程终端

1. 在左侧选中一台服务器。
2. 点击顶部 **终端**。
3. 进入远程终端窗口。

终端窗口中会显示：

- 连接状态
- 目标主机信息
- 当前目录
- 操作系统类型
- 终端类型
- 退出码
- 已执行命令数
- 输出量

## 8.2 终端快捷操作

终端窗口提供常用快捷命令按钮。

### Linux 主机常见快捷项

- 当前目录
- 列表
- 磁盘
- 内存
- 进程
- 失败服务
- 清空视图

### Windows 主机常见快捷项

- 当前目录
- 列表
- 磁盘
- 服务
- 进程
- 清空视图

## 8.3 终端常用功能

终端支持以下功能：

- 手动输入命令并执行
- 中断当前命令
- 自动滚动输出
- 复制
- 全选
- 导出输出日志
- 清空输出
- 重连

命令在终端底部的提示符后直接输入，按 `Enter` 执行；执行中的命令可使用 `Ctrl+C` 中断，输出区仍支持选中文本后复制。

## 8.4 远程文件面板

终端右侧包含远程文件面板，支持：

- 同步到当前终端目录
- 刷新目录
- 浏览远程目录树
- 上传文件或目录
- 下载文件
- 重命名
- 删除
- 新建目录

## 8.5 文件面板使用限制

远程文件面板依赖 SFTP 能力。

因此：

- 如果目标主机只支持 SSH、不支持 SFTP，终端仍可使用
- 但远程文件浏览、上传、下载等功能可能不可用或被禁用

## 9. 脚本管理

“脚本管理”标签页用于管理目标服务器上的脚本文件。

## 9.1 使用前提

进入脚本管理前，请先在左侧选中一台服务器。

如果未选择服务器，界面会提示先选择一台服务器。

## 9.2 主要功能

脚本管理页面支持：

- 选择本地脚本文件夹
- 设置远程目录
- 追加单个脚本文件
- 上传单个脚本
- 加载远程目录结构
- 加载远程脚本列表
- 选择脚本并查看内容
- 编辑脚本内容
- 保存脚本
- 上传整个文件夹

## 9.3 典型使用流程

1. 选中服务器。
2. 切换到 **脚本管理** 标签页。
3. 选择本地脚本文件夹或单个脚本文件。
4. 填写远程目录。
5. 点击 **加载** 或 **加载列表**，查看远程目录情况。
6. 上传脚本或编辑后保存。

## 13. 常见问题

## 13.1 添加服务器时为什么不能保存？

常见原因：

- 主机信息未填写完整
- 还没有执行测试连接
- 测试连接未成功

处理建议：

1. 检查主机地址、端口、用户名是否正确
2. 检查认证方式与密码或私钥是否匹配
3. 先点击 **测试连接**
4. 连接成功后再保存

## 13.2 Windows 服务器为什么连接不上？

当前程序对 Windows 目标机同样使用 SSH 连接。

请确认：

- Windows 服务器已启用 OpenSSH
- SSH 服务已启动
- 防火墙已放通 SSH 端口
- 登录账号允许 SSH 登录

## 13.3 批量安装或批量卸载按钮为什么是灰色的？

通常是因为以下条件未满足：

- 没有选中服务器
- 没有在应用市场中选中应用
- 当前已有批量任务在执行

处理建议：

1. 在左侧多选服务器
2. 在应用市场中点击需要操作的应用卡片，使其进入“已选中”状态
3. 再执行批量操作

## 13.4 终端可以打开，但文件面板无法使用，为什么？

通常表示：

- SSH 可用
- 但 SFTP 不可用，或者当前远程文件能力受限

此时你仍可继续使用终端，但文件上传、下载、重命名、删除等功能可能不可用。

## 13.5 安装资源从哪里来？

程序支持的安装来源以安装配置窗口中的选项为准，常见包括：

- 本地资源
- 在线安装

此外，部分本地资源会从程序的 `Scripts` 目录中自动检测。

## 13.6 为什么某些应用只能本地安装？

不同应用的安装方式由当前程序配置决定。

例如：

- MariaDB 当前界面已明确提示仅支持离线资源安装

因此，如果当前窗口没有显示“在线安装”选项，请按本地资源方式操作。

## 13.7 历史记录和日志有什么区别？

- **历史记录**：更适合查看安装/卸载结果、时间、状态和导出记录
- **日志查看器**：更适合查看程序日志文件和快速排查问题
- **任务详情日志**：更适合查看单次任务的完整执行过程

## 13.8 日志文件在哪里？

默认路径：

`%LocalAppData%\RemoteInstaller\Logs`

如果不确定具体路径，可以直接在程序中点击：

**日志** → **打开文件夹**

## 14. 推荐使用顺序

对于大多数日常使用场景，建议按以下顺序操作：

1. 先添加服务器并测试连接
2. 选中服务器后刷新应用状态
3. 在应用市场中安装或卸载目标应用
4. 通过任务列表查看执行进度
5. 需要排查时打开任务详情或日志查看器
6. 需要运维操作时打开终端
7. 需要脚本或自定义部署时使用脚本管理和自定义应用功能

以上流程可以覆盖当前程序的主要使用场景。
