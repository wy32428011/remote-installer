# RemoteInstaller 图标设计说明

## 1. 背景

当前项目已经具备 Windows 安装器基础链路，但尚未提供项目自己的正式图标资源，导致安装器、应用程序窗口和桌面快捷方式仍缺少统一的品牌识别。为了提升安装体验和产品完整度，需要为 `RemoteInstaller` 设计一套适合 Windows 桌面环境的小尺寸高识别图标，并用于后续接入安装器与应用本体。

## 2. 设计目标

本次图标设计需要同时满足以下目标：

- 体现“远程安装应用”的产品语义
- 风格简洁、专业、偏科技蓝
- 在 Windows 小尺寸图标场景下仍能保持清晰识别
- 适用于安装器图标、应用程序图标、桌面快捷方式图标

## 3. 最终设计方向

### 3.1 核心概念

采用“科技蓝圆角方形底板 + 服务器机柜 + 终端符号 `>_`”的组合图标。

语义映射如下：

- 服务器机柜：表示被远程管理和安装的软件宿主环境
- 终端符号 `>_`：表示远程执行、命令驱动、运维操作
- 圆角方形底板：增强桌面应用属性，适配 Windows 图标常见视觉轮廓
- 科技蓝主色：传达稳定、专业、现代工具感

### 3.2 视觉结构

图标整体结构如下：

- 外层为圆角方形底板
- 中心主体为简化服务器机柜图形
- 在服务器机柜中部或偏右下区域嵌入精简的 `>_` 符号
- 服务器细节仅保留 2 到 3 组横线，不叠加复杂按钮、灯效、连接线或写实阴影

### 3.3 可读性约束

为保证在小尺寸场景下保持识别性，设计时必须遵守以下约束：

- 优先保留圆角方形轮廓
- 优先保留服务器主体块面
- 优先保留 `>_` 终端语义
- 不使用过细描边
- 不堆叠过多内部细节
- 不采用写实金属质感
- 不使用复杂多层投影

## 4. 色彩规范

建议使用以下主色方案：

- 主蓝：`#1E88E5`
- 深蓝：`#1565C0`
- 高光浅蓝：`#64B5F6`
- 前景白：`#FFFFFF`

使用原则：

- 底板允许使用轻微渐变，但整体保持扁平简洁
- 服务器主体与终端符号优先使用白色或浅色
- 不引入额外第二主色，避免视觉噪音

## 5. 输出要求

图标资源后续应至少输出为 `.ico` 文件，并内含以下常用尺寸：

- 16x16
- 24x24
- 32x32
- 48x48
- 64x64
- 128x128
- 256x256

如后续需要保留矢量母版，建议同时保留一份 SVG 源文件用于再导出。

## 6. 使用范围

本图标后续用于以下位置：

1. Windows 安装器图标
2. 应用程序 EXE / 窗口图标
3. 桌面快捷方式图标
4. 开始菜单快捷方式图标

## 7. 当前范围边界

本次仅确认图标设计方向和输出规范，不包含以下内容：

- 创建安装器欢迎页 banner 资源
- 创建品牌宣传插图
- 扩展完整视觉识别系统

这些内容如后续需要，应单独规划。

## 8. 后续实现建议

在实现阶段建议按以下顺序推进：

1. 先生成图标母版
2. 导出多尺寸 `.ico`
3. 接入安装器 `SetupIconFile`
4. 接入应用程序图标配置
5. 验证安装器、桌面快捷方式和程序窗口中的显示效果
